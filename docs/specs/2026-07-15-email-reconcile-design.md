# Дизайн: авто-сверка отправленных писем с Gmail

**Дата:** 2026-07-15
**Статус:** принят, ожидает реализации

## Проблема

Часть outreach-писем оператор отправляет **вручную из папки «Черновики» Gmail**
(куда их кладёт `draft_to_gmail`), а не через MCP-инструмент `approve_email`. Агент
помечает письмо `status='sent'` только при отправке через `approve_email`, поэтому
отправленное вручную остаётся в БД как `status='draft'`. Последствия:

1. Повторный `approve_email` по такому письму отправит адресату **дубль**.
2. Счётчик `emailsSentToday` (предохранитель `emailDailyLimit`) занижен на число
   ручных отправок.
3. `get_email_queue` показывает как ждущие письма, которые уже ушли.

## Решение

Сверять черновики с папкой «Отправленные» Gmail по **получателю + теме** и помечать
совпавшие как `sent`. Тема детерминирована (`Отклик на вакансию «{title}» — {имя}`) и
уникальна на вакансию, поэтому даёт надёжный ключ без изменения схемы БД. Отказ
безопасный: отредактировал тему перед ручной отправкой → просто не сматчится, письмо
останется `draft` (ложного «sent» не возникает).

Отклонённая альтернатива — матч по Message-ID: требует миграции (новая колонка), а Gmail
часто перегенерирует Message-ID при отправке черновика, так что надёжность ниже темы.

## Архитектура

### Новый модуль `src/email/reconcile.ts`

```
reconcileSentDrafts(
  db: Database,
  cfg: Config,
  opts?: { makeClient?: ImapClientFactory; sinceDays?: number },
): Promise<{ reconciled: number; ids: number[] }>
```

Алгоритм:

1. `drafts = repo.getEmailsByStatus(db, "draft")`; если пусто — вернуть `{reconciled:0, ids:[]}`
   **без единого IMAP-захода** (быстрый путь).
2. Нет `SMTP_PASSWORD` — вернуть 0 (IMAP недоступен; не наша задача его требовать здесь).
3. Подключиться IMAP (инъекция `makeClient`, дефолт — `ImapFlow`, как в `gmailDraft.ts`).
4. Найти папку «Отправленные» по `specialUse === "\\Sent"` через `list()`; нет её — вернуть 0.
5. `mailboxOpen(sent)`, `search({ since: <now - sinceDays> })` (по умолчанию 60 дней) →
   UID'ы, `fetchAll(uids, { envelope: true })` → конверты `{to, subject, date}`.
6. Для каждого `draft` найти конверт, где **какой-либо** адрес из `envelope.to` равен
   `draft.to_email` (без учёта регистра) и `envelope.subject` == `draft.subject`
   (точное сравнение). При совпадении:
   `repo.markEmailSent(db, id, sentAt)` (sent_at = дата письма) и
   `repo.setStatus(db, vacancy_id, "applied", { applied_at: sentAt })` — как делает `approveEmail`.
7. `finally { logout() }`. Вернуть число и id сматченных.

### Расширение IMAP-интерфейса

`ImapClientLike` (сейчас `connect/list/append/logout`) дополняется тремя методами,
которые **уже есть у `ImapFlow`**, поэтому дефолтная фабрика (`new ImapFlow(...)`)
удовлетворяет их без обёрток:

- `mailboxOpen(path): Promise<unknown>`
- `search(query): Promise<number[]>` (UID'ы; вызываем с `{ since }`)
- `fetchAll(range, query): Promise<{ envelope?: { subject?: string; to?: {address?: string}[]; date?: Date } }[]>`

`repo.markEmailSent` расширяется необязательным `sentAt` (по умолчанию — сейчас), чтобы
проставлять реальное время отправки из письма.

### Точки вызова (триггер «перед email-действиями»)

- **`approveEmail` (`src/email/approve.ts`)** — главный барьер от дубля. В начале вызывает
  `reconcileSentDrafts`, затем **перечитывает письмо** (`repo.getEmailById`): если оно уже
  `sent` — возвращает `{ error: "уже отправлено вручную (сверено с Gmail), не дублирую" }`.
- **`get_email_queue` (`src/mcp/server.ts`)** — перед выдачей очереди вызывает сверку, чтобы
  показывать только реально ждущие черновики.

Обе точки получают `ImapClientFactory` инъекцией (для фейков в тестах), симметрично
`gmailDraft.ts`.

## Обработка ошибок

- **`approveEmail` — fail-closed:** если сверка бросила (IMAP недоступен), отправку НЕ делаем,
  возвращаем `{ error: "сверка с Gmail не удалась, повтори позже" }`. Цель — не задублить,
  поэтому при неопределённости не отправляем.
- **`get_email_queue` — best-effort:** ошибка сверки логируется (`logger`), очередь отдаётся
  по текущей БД с полем-пометкой `reconcile_failed: true`.
- Счётчик `emailsSentToday` чинится сам: как только черновики помечены `sent`.

## Тестирование (фейк IMAP, без сети — как везде в проекте)

`tests/email-reconcile.test.ts`:

- матч `to`+`subject` → письмо `sent` (с временем из конверта) + вакансия `applied`;
- два черновика на один адрес с разными темами (кейс 2UP) → матчится верный;
- нет совпадения (другая тема) → остаётся `draft`;
- нет `SMTP_PASSWORD` / нет папки «Отправленные» → `{reconciled:0}` без падения;
- сканируются только `status='draft'` (уже `sent`/`rejected` не трогаются).

`tests/email-send.test.ts` (дополнить): `approveEmail` со сверкой, где письмо уже ушло →
возвращает ошибку-отказ, `mailer.send` НЕ вызывается.

## Вне скоупа (YAGNI)

- Матч по Message-ID и связанная миграция.
- Разбор ответов/тредов (это отдельная задача мониторинга откликов).
- Автоотправка чего-либо: сверка только переводит `draft → sent`, ничего не шлёт.
