## 1. Обзор

### Проблема

Активный поиск работы на hh.ru требует ежедневной ручной рутины: просмотр десятков
вакансий, оценка релевантности, изучение компании, написание сопроводительного письма,
отклик. На качественный отклик уходит 20–40 минут; при цели 10 откликов в день это
3–6 часов ежедневно. Без персонализированных писем конверсия в приглашения низкая
(статистика профиля: 26 показов, 5 просмотров, 0 приглашений за неделю пассивного режима).

### Решение

`hh-agent` — локальный автономный агент, который дважды в день сам находит свежие
вакансии по профилю, отсеивает нерелевантные жёсткими фильтрами и LLM-скорингом,
изучает компанию через Perplexity, пишет персонализированное сопроводительное письмо
(Claude) и откликается через браузер пользователя. Управляется из Claude Desktop
как MCP-сервер. Участие пользователя — только контроль отчётов и разбор приглашений.

## 2. Цели и метрики

### Бизнес-цели

- Получать приглашения на собеседования без ежедневного ручного труда
- Довести долю времени пользователя до ≤ 15 мин/день (контроль отчётов)

### Метрики успеха (KPI)

| Метрика                                                | Текущее значение          | Цель      | Срок            |
| ------------------------------------------------------ | ------------------------- | --------- | --------------- |
| Релевантных откликов в день                            | 0 (вручную, эпизодически) | 10        | с первой недели |
| Конверсия отклик → просмотр резюме                     | —                         | ≥ 40%     | 4 недели        |
| Конверсия отклик → приглашение                         | 0                         | ≥ 5%      | 4 недели        |
| Время пользователя в день                              | 3–6 ч (при ручном режиме) | ≤ 15 мин  | с первой недели |
| Ложные отклики (нерелевантные, по оценке пользователя) | —                         | ≤ 1 из 10 | 2 недели        |

## 3. Функциональные требования

### MVP (v1.0)

- [ ] FR-001: Система должна искать вакансии на hh.ru по настраиваемому списку поисковых запросов (язык запросов hh: `OR`, кавычки), регион и параметры — из конфига
- [ ] FR-002: Система должна отсеивать вакансии жёсткими фильтрами: зарплатная вилка, формат работы (офис Москва / гибрид / удалёнка), требуемый опыт, свежесть публикации, чёрный список работодателей
- [ ] FR-003: Система должна дедуплицировать вакансии по id и никогда не откликаться на одну вакансию дважды (уникальный констрейнт в БД)
- [ ] FR-004: Система должна оценивать соответствие вакансии резюме через Claude API: score 0–100 + причины + red flags; в пайплайн проходят вакансии со score ≥ порога (стартово 65)
- [ ] FR-005: Система должна собирать справку о компании через Perplexity API (продукт, стек, новости, репутация) с кэшированием по работодателю
- [ ] FR-006: Система должна генерировать персонализированное сопроводительное письмо через Claude API: официальный тон, от лица ИИ-агента (агент явно представляется), 120–180 слов, опора на резюме, вакансию и справку о компании
- [ ] FR-007: Система должна отправлять отклик с письмом через браузер (Playwright, persistent-профиль с залогиненной сессией hh.ru)
- [ ] FR-008: Система должна соблюдать лимит 10 откликов в сутки, паузы 3–7 минут с джиттером между откликами и рандомизированные задержки во всех действиях браузера (навигация, скролл, клики, ввод текста), включая случайный порядок обработки вакансий и джиттер старта сессий
- [ ] FR-009: Система должна запускаться по расписанию (2 сессии в день) встроенным планировщиком
- [ ] FR-010: Система должна предоставлять MCP-инструменты: `status`, `run_now` (с параметром `mode: live | dry_run`), `pause`, `resume`, `get_report`, `get_queue`, `get_vacancy`, `set_filters`, `blacklist_add` / `blacklist_remove`
- [ ] FR-011: Система должна поддерживать режим dry-run: полный пайплайн без финальной отправки отклика; первые запуски — принудительно dry-run
- [ ] FR-012: Система должна при капче, разлогине или 3 подряд ошибках парсинга останавливать автопилот и уведомлять пользователя (macOS-уведомление); капчу система не решает, но оставляет окно браузера открытым для ручного прохождения; после снятия капчи (автодетект в течение 30 мин или команда `resume`) сессия продолжается с места остановки
- [ ] FR-013: Система должна логировать все запросы и ответы к Claude и Perplexity в БД (таблица `llm_calls`: провайдер, назначение, запрос, ответ, токены, стоимость)
- [ ] FR-014: Система должна формировать дневной отчёт: найдено / отфильтровано / оценено / откликнулись / ошибки / затраты на LLM

## 4. Нефункциональные требования

| Категория          | Требование                                                                                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------ |
| Производительность | Одна сессия пайплайна ≤ 60 мин (с учётом пауз между откликами)                                                                 |
| Стоимость          | LLM-затраты ≤ $50/мес; фактические затраты видны в `llm_calls`                                                                 |
| Безопасность       | API-ключи в `.env` (вне git); сессия hh.ru — в локальном браузерном профиле; письма явно подписаны как отправленные ИИ-агентом |
| Надёжность         | Ошибки API — ретраи с экспоненциальным бэкоффом; отказ любого этапа не приводит к повторному отклику                           |
| Наблюдаемость      | Все LLM-вызовы, запуски и статусы вакансий — в SQLite; структурированные логи                                                  |
| Локализация        | Интерфейс конфигурации и письма — русский                                                                                      |

## 5. Ограничения и допущения

### Ограничения

- Автоматизация браузера — серая зона правил hh.ru: обязательны человекоподобный темп, дневной лимит, немедленный стоп при капче (см. [ADR-001](./adr/001-канал-откликов.md))
- Mac пользователя должен быть включён в моменты запусков по расписанию
- Лимит hh.ru — 200 откликов/сутки; наш лимит 10/сутки, запас 20×

### Допущения

- Сессия hh.ru в persistent-профиле живёт неделями без повторного логина
- Вёрстка hh.ru меняется нечасто; селекторы изолированы в одном модуле
- Резюме на hh.ru опубликовано (текущее: «Системный аналитик/ИИ-разработчик»); обновление резюме — вне зоны ответственности агента
- 10 качественных откликов в день достаточно для целевых конверсий

### Вне скопа (out of scope)

- Решение или обход капч — принципиально
- Официальный API hh.ru (возможный переезд — см. ADR-001)
- Другие джоб-борды (LinkedIn, Хабр Карьера)
- Ответы на сообщения работодателей (v2.0)
- Изменение резюме на hh.ru
