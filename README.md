# hh-agent

Автономный ИИ-агент для откликов на вакансии hh.ru. Локальный MCP-сервер на
TypeScript: браузер (Playwright) ищет вакансии, LLM (Claude + Perplexity)
скорит их и пишет сопроводительные письма, встроенный планировщик запускает
сессии по расписанию, управление и мониторинг — через MCP-инструменты из
Claude Desktop.

Подробности архитектуры, схемы БД и AI-спецификации — в [docs/](./docs/README.md).

## Требования

- Node.js ≥ 20 (проверено на v24)
- Ключи API: `ANTHROPIC_API_KEY`, `PERPLEXITY_API_KEY`
- Аккаунт на hh.ru (логин выполняется вручную один раз, см. ниже)
- macOS (для автозапуска через launchd; сам агент кросс-платформенный)

## Установка

```bash
npm i
npx playwright install chromium
cp .env.example .env
```

Открыть `.env` и вписать реальные ключи:

```
ANTHROPIC_API_KEY=sk-ant-...
PERPLEXITY_API_KEY=pplx-...
```

### Резюме

Агент читает резюме из файла (markdown) для скоринга и генерации писем.
Путь по умолчанию — `~/.hh-agent/master.md`. Либо:

- положить файл резюме по этому пути, **либо**
- указать свой путь через `resumePath` в `~/.hh-agent/config.json` (создаётся
  автоматически при первом запуске со значениями по умолчанию — см.
  `src/config.ts`).

### Первый логин на hh.ru

Браузерный профиль хранится в `~/.hh-agent/profile` (persistent context,
`headless: false`) — окно Chromium открывается видимо. Логиниться нужно
один раз вручную:

```bash
npx tsx scripts/probe.ts
```

Скрипт откроет окно браузера на hh.ru — войти в аккаунт в открывшемся окне
(логин/пароль или SMS). Сессия сохранится в профиле, повторный вход не
понадобится. Скрипт заодно проверит актуальность селекторов на реальных
страницах поиска и вакансии и сохранит HTML-фикстуры в `tests/fixtures/`.

## Запуск

Режим разработки (без сборки):

```bash
npm run dev
```

Продакшн-сборка:

```bash
npm run build
npm start
```

После старта в консоли появится строка вида `[mcp] http://localhost:7010/mcp`
и `[scheduler] armed: ...` — сервер и планировщик подняты.

## Подключение MCP к Claude Desktop

В конфиге Claude Desktop (`~/Library/Application Support/Claude/claude_desktop_config.json`)
добавить сервер по адресу `http://localhost:7010/mcp` (streamable HTTP,
без авторизации — сервер слушает только `127.0.0.1`). После перезапуска
Claude Desktop в списке инструментов появятся 10 команд hh-agent:

| Инструмент | Назначение |
|---|---|
| `status` | Текущее состояние: режим, пауза, отклики за сегодня, лимит, очередь, порог |
| `run_now` | Запустить сессию сейчас (опционально `mode`: `live`/`dry_run`) |
| `pause` | Поставить автопилот на паузу |
| `resume` | Снять с паузы |
| `get_report` | Отчёт за день (YYYY-MM-DD, по умолчанию сегодня) |
| `get_queue` | Очередь на отклик: score и текст письма по каждой вакансии |
| `get_vacancy` | Вакансия целиком по id |
| `set_filters` | Частичный патч конфига (например, `{"mode":"live"}`) |
| `blacklist_add` | Добавить работодателя в чёрный список |
| `blacklist_remove` | Убрать работодателя из чёрного списка |

## Автозапуск через launchd

Автозапуск поднимает планировщик, который дважды в день (`0 10 * * *` и
`30 15 * * *` по умолчанию, см. `schedule` в конфиге) открывает окно браузера
и обращается к LLM. **Устанавливать имеет смысл только после калибровочной
недели** (см. ниже) — до этого в конфиге и `.env` должно быть всё настроено
и проверено вручную через `run_now`.

Чек-лист перед установкой:

- [ ] `.env` заполнен реальными ключами
- [ ] выполнен первый логин на hh.ru (профиль в `~/.hh-agent/profile` валиден)
- [ ] резюме лежит по `resumePath` из `~/.hh-agent/config.json`
- [ ] `npm run build` прошёл чисто
- [ ] калибровочная неделя в `dry_run` пройдена и метрики устраивают (см. ниже)

Путь к `node` в готовом plist — `/opt/homebrew/bin/node` (Homebrew на Apple
Silicon). Если ваш Node из nvm или другого места — проверьте `which node`
и поправьте `ProgramArguments` в `launchd/com.aleksandr.hh-agent.plist`.

Установка:

```bash
npm run build
cp launchd/com.aleksandr.hh-agent.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.aleksandr.hh-agent.plist
tail -f /tmp/hh-agent.log   # ждём [mcp] и [scheduler] armed
```

Остановить и выгрузить автозапуск:

```bash
launchctl unload ~/Library/LaunchAgents/com.aleksandr.hh-agent.plist
```

Логи: `/tmp/hh-agent.log` (stdout), `/tmp/hh-agent.err` (stderr).

## Калибровочная неделя (dry-run)

По умолчанию конфиг стартует в `mode: "dry_run"` — агент ищет вакансии,
скорит их, пишет письма, но реальных откликов не отправляет. Запускать
сессии вручную из Claude Desktop:

```
run_now            # без mode — возьмёт dry_run из конфига
get_queue          # посмотреть очередь: score и письма глазами
get_report         # дневная статистика
```

Цели калибровки (полностью — в [docs/ai-spec.md](./docs/ai-spec.md#оценка-качества-evals)):

| Метрика | Цель |
|---|---|
| Согласие с решением скоринга | ≥ 85% (разметка ~50 вакансий через `get_queue`) |
| Письма без правок | ≥ 8 из 10 |
| Галлюцинации в письмах | 0 (каждый факт письма сверяется с резюме и справкой) |
| Ложный пропуск сильных вакансий | ≤ 1 из 20 (просмотр отсеянных со score 50–64) |

По итогам калибровки — скорректировать `scoreThreshold` и промпты
(`src/llm/prompts.ts`), зафиксировать версию.

## Переключение в live

Только после того как калибровочная неделя пройдена и метрики выше
достигнуты — осознанно включить реальные отклики:

```
set_filters {"mode":"live"}
```

Перед этим стоит ещё раз проверить `dailyLimit` и `schedule` в конфиге —
это то, что реально ограничивает частоту откликов в бою. Откат в dry-run —
тем же вызовом с `"mode":"dry_run"`.
